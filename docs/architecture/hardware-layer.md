# 硬件层

## 概述

Open-RetroSight 支持多种硬件方案，以满足不同场景的需求和预算。

## 硬件方案对比

| 方案 | 成本 | 算力 | 适用场景 |
|-----|------|------|---------|
| 极客版 | < 300元 | 中等 | DIY、原型验证 |
| 利旧版 | ~0元 | 较高 | 快速试用、成本敏感 |
| 工业版 | 500-1000元 | 高 | 生产环境、多设备 |

## 方案A：极客版

### 配置
- **计算单元**：Raspberry Pi Zero 2W / 香橙派
- **摄像头**：USB 工业摄像头
- **总成本**：< 300元

### 推荐硬件

#### 计算单元
| 型号 | 价格 | 特点 |
|-----|------|------|
| Raspberry Pi Zero 2W | ~150元 | 体积小、功耗低、生态好 |
| 香橙派 Zero2 | ~100元 | 性价比高、国产 |
| Raspberry Pi 4B | ~400元 | 算力强、接口丰富 |

#### 摄像头
| 类型 | 价格 | 特点 |
|-----|------|------|
| USB 工业摄像头 | 50-150元 | 稳定、支持手动对焦 |
| CSI 摄像头模块 | 30-80元 | 体积小、延迟低 |

### 优点
- 成本低
- 社区支持好
- 易于定制

### 缺点
- 需要一定动手能力
- 工业环境适应性一般

## 方案B：利旧版

### 配置
- **计算单元**：退役 Android 手机
- **摄像头**：手机自带摄像头
- **成本**：几乎为零

### 要求
- Android 7.0 以上
- 摄像头像素 500万以上
- 支持 WiFi

### 优点
- 成本极低
- 摄像头素质高
- 自带 WiFi 和电池

### 缺点
- 需要开发 Android App
- 散热可能成问题
- 长期稳定性未知

## 方案C：工业版

### 配置
- **图像采集**：ESP32-CAM（仅传输图像）
- **计算单元**：边缘服务器（集中处理）
- **成本**：500-1000元

### 架构
```
ESP32-CAM ─┐
ESP32-CAM ─┼─→ 边缘服务器 ─→ 云端
ESP32-CAM ─┘
```

### 推荐硬件

#### ESP32-CAM
- 价格：~30元
- 特点：超低成本、WiFi 内置
- 功能：仅负责图像采集和传输

#### 边缘服务器
| 型号 | 价格 | 特点 |
|-----|------|------|
| Intel NUC | 1500-3000元 | 性能强、稳定 |
| Jetson Nano | 800-1500元 | GPU加速、AI优化 |
| 工控机 | 2000-5000元 | 工业级、高可靠 |

### 优点
- 可扩展性强
- 集中管理
- 工业级可靠性

### 缺点
- 初始成本较高
- 部署复杂度高

## 辅助配件

### 必备配件

| 配件 | 用途 | 价格 |
|-----|------|------|
| 环形补光灯 | 消除阴影、稳定光线 | 20-50元 |
| 万向支架 | 固定摄像头位置 | 30-80元 |
| 防尘外壳 | 保护设备 | 50-100元 |

### 可选配件

| 配件 | 用途 | 价格 |
|-----|------|------|
| 遮光罩 | 避免反光 | 10-30元 |
| 散热风扇 | 高温环境散热 | 10-30元 |
| UPS电源 | 断电保护 | 100-300元 |

## 选型建议

### 原型验证
推荐：**极客版**
- 快速搭建
- 成本可控
- 便于调试

### 成本敏感
推荐：**利旧版**
- 零成本试用
- 验证可行性

### 生产环境
推荐：**工业版**
- 高可靠性
- 易于维护
- 可扩展
